Feature: Авторизация доступа к карточкам
  Scenario: Пользователь не может удалить чужую карточку
    Given Пользователь "User A" имеет карточку с ID "card-123"
    And   Пользователь "User B" авторизован в системе
    When  "User B" отправляет запрос DELETE на /cards/card-123
    Then  Сервер должен вернуть ответ со статусом 403 Forbidden
Feature: Производительность API получения карточек
  Scenario: API быстро отдает список карточек
    Given В базе данных существует 1000 карточек для одного пользователя
    When  Запускается нагрузочный тест с 50 RPS на эндпоинт GET /cards
    Then  p99 времени ответа должен быть меньше или равен 250 мс
Feature: Валидация перемещения карточек
  Scenario: Нельзя переместить карточку в несуществующую колонку
    Given Существует карточка с ID "card-456" в колонке "TODO"
    When  Пользователь отправляет запрос PATCH /cards/card-456/move с телом {"column": "NonExistentColumn"}
    Then  Сервер должен вернуть ответ со статусом 422 Unprocessable Entity
